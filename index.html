STM32的I2S总线问题（未解决）

  • 关于I2S的接线问题：第38章 I2S—音频播放与录音输入—零死角玩转STM32-F429系列 - 野火_firege - 博客园 (cnblogs.com)
    • 以PCM1808接STM32F103RCT6的I2S2为例：
      • DOUT→SD（(Serial Data)：串行数据线，用于发送或接收两个时分复用的数据通道上的数据(仅半双工模式)，如果是全双工模式，该信号仅用于发送数据。）→PB15
      • LRCK→ WS(Word Select)：字段选择线，也称帧时钟(LRC)线，表明当前传输数据的声道，不同标准有不同的定义。WS线的频率等于采样频率(FS)。→PB12
      • SCK/BCK→CK(Serial Clock)：串行时钟线，也称位时钟(BCLK)，数字音频的每一位数据都对应有一个CK脉冲，它的频率为：2采样频率量化位数，2代表左右两个通道数据。→PB13
  • 另外还有： ext_SD(extend Serial Data)：扩展串行数据线，用于全双工传输的数据接收。→PB14

MYSQL的学习

踩过的坑

  • 在配置mysql的时候踩过的坑还是挺多的，包括但不限于：
    • 开始装了一堆奇怪的服务，结果server的服务在终端上意外地不太好用，心态炸裂
    • 曾经没有记住命令 mysqld --initialize --console这个命令产生时的密码，导致的后果是——登录不了，需要各种方法绕，比如：(128条消息) Windows下忘记MySQL数据库root用户密码解决办法_Random-CSDN博客_windows忘记mysqlroot密码
    • 在开始服务的时候没有用上管理员模式，导致结果欠佳
    • 关于服务无法启动的问题：(128条消息) net start mysql无法启动MySQL_咕咕咕的下班时间-CSDN博客_mysql net start

最终的选择

  • 最终选择的结果为：命令行下的MySQL当然这种情况的话，在配置的时候也会有不少麻烦，最后是看了这篇文章：(128条消息) Mysql压缩包版的安装教程_Z2Min_-CSDN博客_mysql压缩包怎么安装————讲的东西还比较详细，而且也是基础范围内的
  • 教程一如既往还是看廖雪峰的

  • 关于可视化工具的选择：选择了官方的MySQLWorkbench作为可视化工具，直接上官网下载即可
    • 但是把内容汉化了，却没有保存，参考的gitee仓库为：MySQL Workbench 汉化: MySQL Workbench 汉化 (gitee.com)
